  const [proofFileId, setProofFileId] = useState(null);

## Key Points

1. **No Multer** - Backend uses `express.raw()` to receive raw binary data
2. **Fastest Upload** - Single HTTP request with raw bytes (no multipart overhead)
3. **Progress Tracking** - XHR `upload.onprogress` for real-time percentage
4. **Cancelable** - User can cancel upload mid-way
5. **Auto-cleanup** - Old file deleted when replaced
6. **Presigned URLs** - S3 URLs are temporary (1 hour) for security
7. **Returns fileId only** - `onChange` callback receives MongoDB ObjectId string
